# Technical Architecture Documentation: Soil Nutrition Backend

This document outlines the high-level architecture, technology stack, and component interactions of the Soil Nutrition Backend.

## 1. System Overview

The Soil Nutrition Backend is a Flask-based RESTful API designed to provide machine learning-driven agricultural insights. The system coordinates data from soil tests, integrates with pre-trained models for crop and yield prediction, and manages historical data for analytics dashboards.

### High-Level Architecture
- **Web Layer**: Flask with Blueprints for modular routing.
- **Business Logic**: Service-oriented layer for ML predictions and recommendations.
- **Data Layer**: SQLAlchemy ORM with SQLite (for development) and Marshmallow for validation/serialization.
- **ML Integration**: CatBoost and Scikit-learn (Pickle) model loading and inference.

## 2. Technology Stack

- **Framework**: [Flask](https://flask.palletsprojects.com/) (Python)
- **Database**: [SQLAlchemy](https://www.sqlalchemy.org/) (ORM) / SQLite
- **Validation**: [Marshmallow](https://marshmallow.readthedocs.io/) (Data schemas)
- **Machine Learning**: 
    - [CatBoost](https://catboost.ai/) (Yield prediction)
    - [Scikit-learn](https://scikit-learn.org/) / [Pickle](https://docs.python.org/3/library/pickle.html) (Crop prediction)
- **Authentication**: JWT-based (JSON Web Tokens) with decorator-based route protection.
- **Development Tools**: [dotenv](https://pypi.org/project/python-dotenv/) for configuration, [logging](https://docs.python.org/3/library/logging.html) for monitoring.

## 3. Core Components

### `app/api/` (Controllers)
Handles HTTP requests and responses. It uses Flask Blueprints to isolate features:
- `auth.py`: User registration, login, and authentication middleware.
- `prediction.py`: Endpoints for generating and saving crop/yield predictions.
- `analytics.py`: Data aggregation for dashboard statistics and historical trends.

### `app/services/` (Business Logic)
Encapsulates complex logic away from the API layer:
- `crop_service.py`: Manages ML model loading (Pickle/CatBoost), feature engineering, and inference logic.
- `recommendation_service.py`: Rules-based engine for generating agricultural advice based on predicted crops and soil data.

### `app/models/` (Data Persistence)
Defines the database schema using SQLAlchemy:
- `user.py`: User profiles and authentication data.
- `prediction.py`: Historical logs of nutrient data, geographic locations, and harvest outcomes.

### `app/schemas/` (Serialization & Validation)
Uses Marshmallow to bridge the gap between JSON requests and Python objects:
- `prediction_schema.py`: Stricly validates soil parameter ranges (e.g., pH 0-14, Nitrogen 0-140) before they reach the ML model.

## 4. Machine Learning Pipeline

### Model Loading (Singleton)
To prevent memory overhead, models are loaded using a **Singleton** pattern in the `CropPredictionService`. Models are loaded from the `models/` directory once and cached for subsequent requests.

### Feature Engineering
The backend performs real-time feature engineering for yield prediction:
- **Soil Fertility Index (SFI)**: Sum of Nitrogen, Phosphorus, and Potassium.
- **NP Ratio**: Nitrogen to Phosphorus ratio.
- **Climate Index**: Rainfall multiplied by Temperature.

## 5. Security & Authentication

- **JWT Integration**: Upon successful login, the server issues a JWT.
- **Route Guarding**: The `@require_auth` decorator is applied to sensitive endpoints, which validates the token and injects the `current_user` object into the function scope.
- **Validation Layers**: Multiple layers of validation ensure that malicious or malformed soil data cannot cause server errors or invalid ML results.
