# ML Training Pipeline Documentation: `train.py`

This document provides a detailed explanation of the machine learning training pipeline implemented in `train.py`. This script is responsible for training the **Yield Prediction** model using the CatBoost algorithm.

---

## 1. Overview
The training pipeline takes historical agricultural data from Nigeria, performs feature engineering, and trains a Gradient Boosted Decision Tree (GBDT) model (CatBoost) to predict crop yield in `kg/ha`.

### Key Technologies
- **CatBoost**: Chosen for its superior handling of categorical variables (like `state`, `crop_type`, `soil_type`).
- **SHAP**: Used for model interpretability (explaining how each feature affects the prediction).
- **Scikit-learn**: Used for cross-validation and performance metrics.

---

## 2. Implementation Details

### 📥 Data Loading (`load_and_inspect_data`)
- **Action**: Loads an Excel file from `data/nigeria_agri_yield_cleaned-updated.xlsx`.
- **Validation**: Checks for the existence of the target column (`yield_kg_ha`) and handles missing values.

### 🛠️ Feature Engineering (`engineer_features`)
The script creates three "Derived Features" to help the model better understand the relationship between soil and climate:
1.  **Soil Fertility Index (SFI)**: `soil_nitrogen + soil_phosphorus + soil_potassium`. A high-level measure of soil health.
2.  **NP Ratio**: `soil_nitrogen / soil_phosphorus`. Important for understanding nutrient balance.
3.  **Climate Index**: `rainfall_mm * temperature_C`. Captures the combined effect of heat and moisture on growth.

### 📊 Data Preparation (`prepare_data`)
- Splits the dataset into Features (`X`) and Target (`y`).
- Automatically detects **Categorical Columns** (text-based) and **Numeric Columns**.
- Saves the preparation metadata to `data_prep_info.txt` for reference.

### 🔄 Cross-Validation Strategy (`train_cv_loop`)
The model is evaluated using **5-Fold Stratified Cross-Validation**:
- **Stratification**: The folds are split based on `crop_type` to ensure that every fold has a representative mix of all crops.
- **Model Parameters**:
    - `iterations=500`: Number of trees.
    - `learning_rate=0.05`: Step size for optimization.
    - `depth=6`: Complexity of each tree.
    - `Early Stopping`: Training stops if the model stops improving for 150 rounds, preventing overfitting.

---

## 3. Evaluation Metrics

The script calculates and prints:
- **RMSE (Root Mean Square Error)**: Measures the average magnitude of the error (penalizes large errors).
- **MAE (Mean Absolute Error)**: Average error in kg/ha.
- **R² (R-Squared)**: How well the model explains the variance in the data (1.0 is perfect).
- **MAPE (Mean Absolute Percentage Error)**: Average percentage difference between prediction and reality.

---

## 5. Execution Flow (Layman's Terms)
1. **Gather**: Pull in the Excel spreadsheet.
2. **Transform**: Calculate new metrics like "Soil Fertility" that make sense to farmers.
3. **Practice**: Train the AI on 4/5ths of the data and test it on the remaining 1/5th. Repeat this 5 times to be sure the AI isn't just "guessing."
4. **Final Exam**: Once we are happy with the accuracy, train the AI on the *entire* dataset.
5. **Save**: Lock the AI's "knowledge" into the `.cbm` file and save the charts to show the humans what the AI learned.
